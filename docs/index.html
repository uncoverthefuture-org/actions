UActions

Local Container Deployment with Podman & Traefik

Overview

What is UActions?

UActions is a lightweight, local container deployment system powered by Podman and Traefik. It can be used locally or deployed to remote servers via SSH.

Features

  • Local Domain Routing - Deploy apps to custom local domains (e.g., tea.mydomain.pc)
  • Automatic Container Management - Podman containers with automatic builds
  • Traefik Integration - Built-in reverse proxy with Let's Encrypt support
  • File Watching - Auto-deploy when you add new artifacts
  • Two Modes: Local (Podman on localhost) or Remote (SSH to server)
  • Lightweight - Minimal resource usage, designed for local development

Quick Example

# Install globally
npm install -g @uncver/actions

# Initialize
uactions init --domain mydomain.pc

# Deploy an app
uactions deploy my-app
Setup

Getting Started

Prerequisites

  • macOS or Linux
  • Podman installed
    • macOS: brew install podman
    • Linux: sudo apt-get install podman
  • Node.js 18+

Installation

# Using npm
npm install -g @uncver/actions

# Or using yarn
yarn global add @uncver/actions

# Or run directly with npx
npx @uncver/actions init --domain mydomain.pc

Initialize UActions

uactions init --domain mydomain.pc --yes

This creates the necessary directories and configures Traefik for local domain routing.

Config

Artifact Configuration

Deployments are configured using artifact.json files in your ~/uactions/ directory.

Example Configuration

{
  "version": "1.0.0",
  "name": "My Application",
  "source": {
    "url": "https://github.com/user/repo.git",
    "ref": "main"
  },
  "domain": {
    "subdomain": "myapp",
    "public": false
  },
  "container": {
    "port": 8080,
    "dockerfile": "./Dockerfile",
    "memory": "512m",
    "cpu": 0.5
  },
  "deploy": {
    "mode": "local"
  }
}

Configuration Options

  • version - Semantic version (required)
  • name - Application name (required)
  • source.url - Git repository URL (required)
  • source.ref - Git branch/tag (default: main)
  • domain.subdomain - Local subdomain (required)
  • domain.public - Enable public HTTPS (default: false)
  • container.port - Container port (default: 8080)
  • container.dockerfile - Dockerfile path
  • container.memory - Memory limit (e.g., "512m")
  • container.cpu - CPU limit (e.g., 0.5)
Usage

Deployment Guide

Local Deployment

# Deploy an artifact
uactions deploy my-app

# Or deploy from a specific file
uactions deploy --file /path/to/artifact.json

Remote Deployment

# Deploy via SSH
uactions deploy my-app --remote user@server.com

Auto-Deploy with Watch Mode

# Watch for changes and auto-deploy
uactions watch

When watch mode is active, any new artifacts added to ~/uactions/ will be automatically deployed.

How It Works

  1. File Watcher monitors ~/uactions/ for artifact.json files
  2. When detected, source is pulled to a temp directory
  3. Docker image is built with Podman
  4. Container is started with Traefik labels
  5. Domain is added to /etc/hosts
  6. Traefik routes traffic from subdomain.your-domain.pc to the container
Reference

CLI Commands

uactions init

uactions init [options]

Options:
  -d, --domain   Set custom base domain (e.g., mydomain.pc)
  -e, --email     Email for Let's Encrypt (enables HTTPS)
  --public               Enable public domain access with Let's Encrypt
  -y, --yes              Skip confirmation prompts

uactions deploy

uactions deploy [name] [options]

Options:
  -s, --source     Source URL to deploy
  -d, --domain     Subdomain for deployment
  -p, --port      Container port (default: 8080)
  --file          Path to artifact.json file
  --create-pr           Create a GitHub PR with Dockerfile

uactions watch

uactions watch

Watch for file changes and auto-deploy.

uactions list

uactions list

List all deployments/artifacts.

uactions status

uactions status

Show system status (Podman, Traefik, deployments).
